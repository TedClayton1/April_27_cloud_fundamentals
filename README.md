# April 27 Cloud Fundamentals – Git Workflow Project

## 📌 Overview

This project demonstrates a basic Git and GitHub workflow, including repository setup, branching, file creation, and pull request (PR) collaboration. The goal was to simulate a real-world development process using version control and code review practices.

---

## 🛠️ Project Setup

The following steps were completed to initialize the project:

1. Created a GitHub repository
2. Cloned the repository to a local machine
3. Created project files:

   * `notes.txt`
   * `index.html`
   * `app.py`

---

## 🔀 Git Workflow

### 1. Create a Feature Branch

A new branch was created to isolate changes from the main branch:

```bash
git checkout -b add-my-files
```

---

### 2. Make Changes

Added content to the following files:

* notes.txt
* index.html
* app.py

---

### 3. Stage and Commit Changes

```bash
git add .
git commit -m "Added initial project files"
```

---

### 4. Push Branch to GitHub

```bash
git push origin add-my-files
```

---

## 🔁 Pull Request Workflow

1. Opened a Pull Request (PR) from `add-my-files` → `main`
2. Reviewed code changes using the "Files changed" tab
3. Submitted a review with feedback and approval
4. Merged the PR into the `main` branch

---

## 👥 Collaboration

* Added a collaborator to the repository
* Simulated team workflow by reviewing and merging code via pull requests

---

## 🧠 Key Concepts Learned

* Git branching strategy
* Pull request creation and review
* Code comparison and validation
* Safe merging into the main branch
* Basic collaboration using GitHub

Successfully implemented a full Git workflow from local development to remote merge, demonstrating foundational version control and collaboration practices used in real-world DevOps environments.
